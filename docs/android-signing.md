# Android Signing and RC Builds

This project now supports signed Android release artifacts (`.apk` and `.aab`) for local testing and CI release candidates.

## 1) Create a new upload keystore

Use `keytool` to generate a dedicated upload key:

```bash
keytool -genkeypair \
  -v \
  -keystore furu-upload.keystore \
  -alias furu-upload \
  -keyalg RSA \
  -keysize 4096 \
  -validity 3650
```

Keep this file private and back it up securely.

## 2) Local release signing inputs

Release signing is read from either:

- `src-tauri/gen/android/app/tauri.properties` (`tauri.android.signing.*` keys), or
- environment variables (recommended):
  - `ANDROID_KEYSTORE_PATH`
  - `ANDROID_KEYSTORE_PASSWORD`
  - `ANDROID_KEY_ALIAS`
  - `ANDROID_KEY_PASSWORD`

Example local shell setup:

```bash
export ANDROID_KEYSTORE_PATH="$PWD/furu-upload.keystore"
export ANDROID_KEYSTORE_PASSWORD="***"
export ANDROID_KEY_ALIAS="furu-upload"
export ANDROID_KEY_PASSWORD="***"
```

## 3) Build signed Android artifacts locally

From repo root:

```bash
devenv shell
pnpm tauri android build --target aarch64
```

Expected outputs:

- APK: `src-tauri/gen/android/app/build/outputs/apk/**/release/*.apk`
- AAB: `src-tauri/gen/android/app/build/outputs/bundle/**/release/*.aab`

Notes:

- `.apk` is for direct install (sideload/testing).
- `.aab` is for Play Console upload.

## 4) GitHub Actions secrets for CI signing

Set these repository secrets:

- `ANDROID_KEYSTORE_BASE64` (base64 of the keystore file)
- `ANDROID_KEYSTORE_PASSWORD`
- `ANDROID_KEY_ALIAS`
- `ANDROID_KEY_PASSWORD`

To generate base64:

```bash
base64 -w 0 furu-upload.keystore
```

## 5) First-run checklist

- Confirm all four secrets are set.
- Trigger the RC workflow manually.
- Verify uploaded artifacts include both signed APK and AAB.
- Install the APK on a device to validate signing/install path.

## Generated Android caveat

`src-tauri/gen/android` is generated by Tauri tooling. After regeneration, confirm signing settings in:

- `src-tauri/gen/android/app/build.gradle.kts`
